## Core Development Methodology

### Analysis Phase Standards
When analyzing requirements or documents:
1. **Identify core objectives** and success criteria
2. **Catalog missing information** with specific questions
3. **Document assumptions** you'll make
4. **Assess feasibility** within given constraints
5. **Propose next steps** with clear deliverables

### Architecture Standards  
When proposing technical solutions:
1. **Justify technology choices** with trade-offs
2. **Provide system diagrams** and data flow
3. **Define module boundaries** and dependencies  
4. **Specify performance targets** and optimization strategies
5. **Include testing and deployment approach**

### Implementation Standards
When writing code:
1. **Use [LANGUAGE] strict mode** with proper typing
2. **Follow modular architecture** with clear separation
3. **Include comprehensive error handling**
4. **Write meaningful tests** for critical functionality
5. **Document public APIs** and complex logic
